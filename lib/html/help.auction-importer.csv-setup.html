CSV Setup (No longer required)

UPDATE (09/20/2021): The auction importer now works with the raw CSV exported from Caspio. No need to manipulate the file, the import now handles the following:

  • Creates the Item Title by concatenating the LotNumber and Lead columns (i.e. `Lot ${LotNumber}: ${Lead}`).
  • Builds the Item Description by concatenating the item Description, ProvenanceLine, and Condition columns (i.e. `${Description}\n\nPROVENANCE: ${ProvenanceLine}\n\nCONDITION: ${Condition}`).

When updating an auction with Low/High Estimates, Realized Prices, and Highlights, use these column headings:

  • LowEstimate
  • HighEstimate
  • HammerPrice
  • isHighlight

Setup your CSV with the following columns: LotNum, Title, Description, LowEst, HighEst, Realized, Highlight, Categories, Tags, iGavelLotNum, BidsquareLotNum, and LiveAuctioneersID. Here's an example:

Notes

  • When exporting a spreadsheet from Excel, choose "Windows Comma Separated (.csv)" as the file format.
  • iGavel Online Auctions: Add a link to any item's corresponding iGavel auction page by adding column called iGavelLotNum (case-sensitive) to your CSV.
  • LiveAuctioneersID - we're able to specify the Live Auctioneers auction ID at the Item level to allow for mutliday auctions which have multiple Live Auctioneers IDs.
